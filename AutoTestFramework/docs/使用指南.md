## 环境搭建

### 安装Python

确保Python版本 >= 3.8：

```bash
python --version
```

### 安装Allure（可选，用于生成Allure报告）

**macOS:**
```bash
brew install allure
```

**Windows:**
1. 下载Allure：https://github.com/allure-framework/allure2/releases
2. 解压到 `C:\Program Files\allure-2.x.x`
3. 添加到系统PATH

**Linux:**
```bash
sudo apt-add-repository ppa:qameta/allure
sudo apt-get update
sudo apt-get install allure
```

## 编写用例

### 基础用例结构

```python
import allure
from core.base_test import BaseTestCase

@allure.feature("模块名称")
class TestExample(BaseTestCase):

    @allure.title("测试用例标题")
    @allure.severity(allure.severity_level.CRITICAL)
    def test_example(self):
        testcase = {
            "id": "TC001",
            "request": {
                "method": "POST",
                "url": "https://api.example.com/endpoint",
                "headers": {"Content-Type": "application/json"},
                "body": {"key": "value"}
            },
            "validate": [
                {"eq": ["status_code", 200]},
                {"contains": ["body.message", "success"]}
            ]
        }
        response = self.run_api_test(testcase)
        assert response.is_success
```

### 验证规则类型

| 规则 | 说明 | 示例 |
|------|------|------|
| eq | 等于 | `{"eq": ["status_code", 200]}` |
| contains | 包含 | `{"contains": ["body", "keyword"]}` |
| jsonpath | JSONPath提取 | `{"jsonpath": ["$.data.id", "123"]}` |
| schema | JSON Schema验证 | `{"schema": "schemas/response.json"}` |
| response_time | 响应时间 | `{"response_time": 1000}` |

## 运行测试

### 命令行参数

```bash
# 指定环境
python run.py -e test

# 并行执行（4线程）
python run.py -n 4

# 只运行特定标记的用例
python run.py -m "smoke"

# 匹配关键字
python run.py -k "login"

# 失败重试3次
python run.py --rerun 3

# 失败即停止
python run.py -x

# 启用钉钉通知
python run.py --dingtalk
```

```bash
# 实时查看

# 生成静态报告
allure generate reports/report_20240115/allure-results -o allure-report
```

## 钉钉通知

### 配置步骤

1. **创建钉钉机器人**
   - 进入钉钉群 → 群设置 → 智能群助手 → 添加机器人
   - 选择"自定义"机器人
   - 设置安全方式（推荐"加签"）

2. **获取Webhook和Secret**
   - 复制Webhook地址
   - 如果选择了加签，保存Secret

3. **配置环境变量**

```bash
# .env文件
DING_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=xxx
DING_SECRET=SECxxx
```

4. **运行测试时启用通知**

```bash
python run.py --dingtalk
```

### 通知内容

- 测试开始通知：包含环境、用例数、启动时间
- 测试完成通知：包含通过率、耗时、报告链接
- 失败告警：包含失败用例列表（@相关人员）

## 常见问题

### Q1: 安装依赖时速度慢？

**A:** 使用国内镜像源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: Allure命令找不到？

**A:** 检查环境变量配置，或尝试：

```bash
# macOS/Linux
export PATH=$PATH:/path/to/allure/bin

# Windows（PowerShell）
$env:PATH += ";C:\Program Files\allure-2.x.x\bin"
```

### Q3: 钉钉通知发送失败？

**A:** 检查以下几点：
1. Webhook地址是否正确
2. 如果设置了加签，Secret是否正确
3. 机器人是否在群中（不是仅创建）
4. 网络是否能访问钉钉服务器

### Q4: 如何跳过某些用例？

**A:** 使用pytest标记：

```python
@pytest.mark.skip(reason="暂时跳过")
def test_skip(self):
    pass

@pytest.mark.skipif(condition=True, reason="条件满足时跳过")
def test_skipif(self):
    pass
```

### Q5: 如何标记用例优先级？

**A:** 使用Allure severity：

```python
@allure.severity(allure.severity_level.BLOCKER)    # 阻塞
@allure.severity(allure.severity_level.CRITICAL)   # 严重
@allure.severity(allure.severity_level.NORMAL)    # 一般
@allure.severity(allure.severity_level.MINOR)     # 次要
@allure.severity(allure.severity_level.TRIVIAL)   # 轻微
```

### Q6: 如何参数化用例？

**A:** 使用pytest.mark.parametrize：

```python
@pytest.mark.parametrize("username,password,expected", [
    ("admin", "123456", 200),
    ("admin", "wrong", 401),
    ("", "123456", 400),
])
def test_login(self, username, password, expected):
    response = self.api.login(username, password)
    assert response.status_code == expected
```

## 进阶用法

### 自定义Fixture

在 `conftest.py` 中添加：

```python
@pytest.fixture(scope="session")
def db_connection():
    """数据库连接fixture"""
    db = DatabaseManager("mysql", **CONFIG.DB_CONFIG)
    db.connect()
    yield db
    db.close()
```

### 用例依赖

```python
@pytest.mark.dependency(name="login")
def test_login(self):
    pass

@pytest.mark.dependency(depends=["login"])
def test_need_login(self):
    # 只有test_login通过才会执行
    pass
```

### 并发控制

```python
# 限制并发数
python run.py -n 4 --dist=loadfile

# 按类分发（避免同一类用例并行冲突）
python run.py -n 4 --dist=loadscope
```
