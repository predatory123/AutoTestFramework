## 核心模块设计

### 1. RequestEngine - 高可用请求引擎

**职责：** 处理所有HTTP请求，提供熔断、重试、连接池管理

**设计模式：**
- 熔断器模式 (Circuit Breaker)
- 装饰器模式 (Retry Decorator)
- 工厂模式 (Session Factory)

**关键算法：**
```python
# 指数退避重试
delay = min(base_delay * (2 ^ attempt), max_delay)
if jitter:
    delay = delay * (0.5 + random.random())
```

### 2. DingTalkNotifier - 通知服务

**职责：** 发送钉钉消息，支持多种消息格式

**安全机制：**
- 时间戳 + Secret 签名验证
- HmacSHA256 加密
- Base64 + URL Encode

**消息类型：**
- text: 纯文本
- markdown: 富文本表格
- action_card: 带按钮的卡片
- feed_card: 多链接列表

### 3. DataDriver - 数据驱动引擎

**职责：** 统一封装多种数据源

**支持格式：**
- YAML (推荐，支持注释)
- JSON (轻量级)
- Excel/CSV (数据量大时)

**缓存策略：**
- 内存缓存已加载数据
- 基于文件路径+参数的缓存键

## 高可用保障

### 1. 熔断器机制

```
┌─────────┐    失败计数    ┌─────────┐
│  CLOSED │ ────────────> │  OPEN   │
│ (正常)  │   超过阈值     │ (熔断)  │
└────┬────┘               └────┬────┘
     │                         │
     │  超时恢复               │  半开测试
     │  (Half-Open)            │  成功→CLOSED
     ▼                         ▼  失败→OPEN
┌─────────────────────────────────────┐
│           HALF_OPEN               │
│        (探测恢复状态)              │
└─────────────────────────────────────┘
```

### 2. 多级重试策略

| 层级 | 重试次数 | 延迟策略 | 适用场景 |
|------|---------|---------|---------|
| 请求级 | 3次 | 指数退避(1s→2s→4s) | 网络抖动 |
| 连接级 | 3次 | 固定1s | 连接池耗尽 |
| 会话级 | 1次 | 立即 | Token过期 |

### 3. 并发控制

```python
# 信号量控制并发
semaphore = asyncio.Semaphore(max_concurrent)

async def bounded_request(req):
    async with semaphore:
        return await process(req)
```

## 部署建议

### 1. CI/CD集成

```yaml
# Jenkins Pipeline
pipeline {
    stages {
        stage('Test') {
            steps {
                sh 'python run.py -e ${ENV} --dingtalk'
            }
        }
        stage('Report') {
            steps {
                allure([
                    includeProperties: false,
                    jdk: '',
                    properties: [],
                    reportBuildPolicy: 'ALWAYS',
                    results: [[path: 'reports/**/allure-results']]
                ])
            }
        }
    }
}
```

### 2. Docker化

```dockerfile
FROM python:3.9-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .
CMD ["python", "run.py", "-e", "test"]
```

### 3. 定时任务

```bash
# crontab -e
```
